# 字幕说话人与出场花字架构

本项目当前给知识查询使用的字幕入口是：

```text
data/knowledge/video_subtitles/docs/<videoKey>.subtitles.md
```

这些文档按 video key 组织，保留章节/线路、前置选择、前后节点、时间码、说话人和字幕正文。需要理解具体视频内容时，调用方应读取这层文档，不直接读取原始 SRT。

## 原始字幕边界

`zh_GL/*.srt` 是普通字幕源文件。它能证明某句话在某个字幕文件中出现，但原始文件本身不一定提供可靠说话人字段。

`*_extra.srt` 不是说话人轨。它更像画面上的出场花字、身份介绍、地点或章节标签。它可以辅助识别“这个场景出现过谁、画面介绍了什么身份”，但不能证明某句台词是谁说的。

## 可用字幕层

`data/knowledge/video_subtitles/docs/*.subtitles.md` 是已经按 HTML/路线图组装后的 video key 字幕片段。它解决两个问题：

- 同一个原始 SRT 可能被不同视频片段引用，不能按整份 SRT 文件总结。
- 需要理解剧情片段时，应知道前置选择、选项文本、前后节点和真实图顺序，不能只看散乱字幕。

## 使用规则

- 目标视频事实以 `*.subtitles.md` 为准。
- 时间码和说话人用于定位证据，不能随意改写。
- 如果说话人为空或不确定，只能描述为“字幕显示/画面中出现”，不要硬判定说话人。
- `_extra.srt` 只能作为出场标签或身份标签辅助，不进入台词说话人证据。
- 游戏日志里的字幕路径只用于定位 video key 或字幕片段，不直接作为可引用字幕正文。
