# 知识库架构

本项目的知识库保存可追溯的剧情字幕片段、人物/物品档案、关系链、别名导航、解锁线索、视频摘要和证据锚点。调用方提供定位输入或检索目标，知识库返回可引用资料与取证路径。权威使用规则见 [data/knowledge/AGENT_GUIDE.md](../../data/knowledge/AGENT_GUIDE.md)。

## 核心原则

- Markdown 是长期知识资料的主格式。
- JSON、JSONL、CSV 和原始 SRT 只作为构建输入、中间产物、运行输入或校验材料。
- 用户不会提供 `choice key`、`video key`、`profile key`、`tag key` 或 `item id`。
- 当前游戏位置、目标剧情片段或目标资料必须由后台日志、运行状态、存档、HTML state、批处理目标或外层运行器提供的结构化输入解析。
- 知识库用于查事实、补背景、扩展关系和消歧，不替代当前运行状态、目标视频字幕和路线图事实。
- 脚本只负责构建、召回、导航和校验路径；命中人物/词条档案后必须保留完整 Markdown 给模型阅读，不允许用关键词脚本摘句、裁剪正文或替模型判断人物关系。

## 目录结构

```text
    items/*.md
    aliases/*.md
    unlocks/*.md
    reference/
    unresolved/
  video_summaries/
```

## 各层职责

`video_subtitles/docs/*.subtitles.md` 是每个 video key 的字幕片段文档。它保存目标视频的可引用事实，包括章节、线路、前置选择、前后节点、时间码、说话人和正文。需要理解具体视频内容时，不要用原始 SRT 替代这层文档。

`dossiers/characters/` 和 `dossiers/items/` 保存可引用的人物、物品、制度和典故资料。文件名保留中文名和结构 key，正文保留来源、路线语境、解锁来源、关系实体、别名和检索锚点。路线小节表示文本来源语境，不表示人物只存在于该路线。

`dossiers/aliases/` 保存固定别名、称号、旧 key、阶段导航和上下文称谓消歧指南。固定别名用于跳转到正式档案；非固定称谓必须结合说话人、路线、章节和前后文消歧。

`dossiers/unlocks/` 保存档案解锁线索，用来理解资料出现时机和资料页更新过程。解锁线索不能替代档案正文，不能证明玩家当前剧情线，也不能独立推出人物关系或事件事实。

`dossiers/reference/` 保存前作/旧作、旧名溯源和未实装候选等补充资料。它不是当前游戏事实的默认来源；只有查询目标需要这些背景时才按需读取。

`dossiers/unresolved/` 保存只有 key、没有正文的条目。这是补采队列和索引占位，不是事实来源。

`video_summaries/` 保存 AI 生成的派生摘要层。它可以作为补充背景；需要核验目标剧情事实时，仍应回到字幕片段、路线图、档案正文和官方字段。

## 检索顺序

1. 外层输入：后台运行状态、批处理目标、测试目标或构建流程传入的结构化目标。
2. 目标 video key 的字幕片段文档，或目标人物/物品的正式档案。
3. Storyline 路径图中的全量顺序、前置选择、兄弟选项、前后视频和合流关系。
4. 同路线已生成前文摘要；摘要层只能作为辅助背景。
5. 人物、物品、别名、称谓消歧和关系链档案。
6. 官方摘要字段。
7. 前作/旧作参考文本。
8. 未恢复正文条目，仅用于补采和排查。

## 运行状态接口

需要当前游戏状态时，外层运行器应传入当前状态。可用字段通常包括：

- 当前章节、线路、节点或 video key。
- 当前选择问题、选项文本、已选项和可选项。
- 当前路径、选择记忆、历史探索记忆和路线记忆。
- 当前数值、关系值和条件判断结果。

Agent 只能消费这些后台来源。没有状态时，应报告缺少运行状态输入，不应要求用户提供 key。

## 档案状态

人物和物品档案按事实可用性分成三类：

- `data/knowledge/dossiers/characters/` 和 `data/knowledge/dossiers/items/`：已经有可引用正文，可用于背景检索、人物/物品消歧和关系扩展。
- `data/knowledge/dossiers/reference/`：前作/旧作参考资料、旧名溯源和未实装候选资料。只有查询目标需要这些背景时才按需读取。
- `data/knowledge/dossiers/unresolved/`：只有定位锚点，没有可引用正文，只用于补采、映射排查和索引占位。

人物档案正文里的“当前游戏档案文本”和“前作/旧作参考文本”是内容边界，不是质量等级。“前作/旧作参考文本”只能解释过往设定，不能覆盖当前游戏文本。
