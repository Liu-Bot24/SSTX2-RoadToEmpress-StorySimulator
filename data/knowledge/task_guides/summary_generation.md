# 视频摘要生成导引

本文说明生成或重写单个 video key 摘要时，如何使用知识库扩展上下文。

## 输入

调用方应提供结构化目标，例如：

- `chapter`
- `videoKey`
- 当前线路或章节
- 当前选择问题、选项文本、已选项和可选项
- 已知路线记忆、数值和条件判断结果

用户不会提供 key。缺少结构化定位信息时，应报告缺少定位输入，不要向用户索要 key。

## 读取顺序

1. 读取 [../AGENT_GUIDE.md](../AGENT_GUIDE.md)。
2. 读取 [../storyline_guide.md](../storyline_guide.md)，确认路线图字段含义。
3. 打开 `../video_subtitles/docs/<videoKey>.subtitles.md`，作为目标视频事实主来源。
4. 根据字幕文档头部、路线图和字幕索引扩展前置选择、前一视频、后续节点和兄弟分支。
5. 按字幕中的人物、物品、称谓和地名读取 `../dossiers/` 下的完整 Markdown 档案。
6. 需要长距离背景时，读取 `../video_summaries/INDEX.md` 和对应摘要文档，但必须先过滤目标自身、路线位置和来源状态。

人物档案文件名通常包含配置 id 和 tag，不要按人名直接拼 `<人名>.md`。需要查人物时，先读取 `../dossiers/characters/README.md`、`../dossiers/aliases/README.md`、`../dossiers/aliases/characters/README.md` 或具体别名 `.md`，确认真实文件名后再打开完整档案；不要把目录路径当作具体文档读取请求。

外层任务输入通常已经给出目标节点、上游选择、同线路前文、后文直接节点和兄弟分支。全量 `storyline_graph_data.json` 只在这些字段不足以判断路线位置、合流或条件节点时用于精准复核；不要反复读取全量路线 JSON。

摘要生成步骤应先完成取证和结构化输出；摘要文件和聚合数据由调用方按项目同步链路写入。

摘要背景只来自任务输入列出的项目内知识库、路线图、字幕片段和已生成摘要。

不要重复读取已读文件；如果已经读到目标字幕、必要导览和必要上下文，立即输出结构化摘要结果。

## 摘要层过滤

- 当前目标视频自己的既有摘要不能作为输入。
- 同路线前文摘要可以作为辅助背景，尤其适合补充长距离前因后果。
- “前文摘要”只指剧情图真实上游路径中的视频摘要；同一选择的兄弟分支不是前一个视频，不能当作本路径前文。
- 摘要层是 AI 生成的派生材料，必须低于目标字幕、路线图和档案正文；不能把它当作一手事实来源。
- 旧摘要、低可信状态、已知错误或来源不明的摘要不能作为背景。
- 后文摘要和兄弟分支摘要只能帮助理解路线位置、去向和差异；不能把后文细节写进当前视频摘要。
- 如果摘要和目标字幕、路线图或档案正文冲突，以目标字幕、路线图和档案正文为准，并记录风险。

## 事实边界

- `event_facts` 只写目标视频已经发生的事实。
- 目标视频事实优先引用 `subtitle:<videoKey>.subtitles.md:<字幕序号或范围>`。
- 官方摘要只在补足字幕无法表达的卡片说明、视觉动作或节点定位时引用，并标注来源。
- 无字幕视频如果使用目标官方摘要，evidence 写 `official_annotation:target`。如果只使用目标元数据，只能使用固定 metadata evidence：`metadata:target:storylineTitle`、`metadata:target:title`、`metadata:target:kind`、`metadata:target:chapterTitle`、`metadata:target:lineTitle`、`metadata:target:videoKey`。不要在 evidence 冒号后追加中文说明、choice 文本或路线图解释。
- 禁止把“无字幕”写成 `subtitle:<videoKey>.subtitles.md:无字幕`；无字幕只能作为 `risk_flags` 或 `context_refs` 说明。
- 人物/词条档案只用于身份、关系、称谓和名词解释，不能写成目标视频已发生事实。
- 前置选择、同路线前文、后文节点和兄弟分支只用于理解上下文、分支差异和路线位置。

## 表达要求

- `display_summary` 和 `detailed_summary` 应写成自然、连贯的剧情复述，说明人物动机、冲突和转折，不要机械改写字幕清单。
- 字幕较短时，可以借助前置选择、相邻节点、官方摘要和必要档案把剧情功能说清楚，但不能补写目标视频没有发生的事实。
- 新世界线中，目标字幕或目标元数据只写“你”“主角”时，主角优先按该线身份写作“伍元照”；不要把女帝线的“伍媚娘”称呼带入新世界线。若目标字幕明确使用其他称呼，以目标字幕为准。
- 使用游戏内名称。官方摘要若残留历史原名，输出时必须改回游戏内名称，例如李治→礼治、李泰→礼泰、李弘→礼弘、武媚娘→伍媚娘。
- 输出 JSON 字符串内不要使用英文双引号；引用称呼、标题或概念时使用中文书名号、中文引号或单引号，避免破坏 JSON。
- `event_facts` 仍保持证据化、可核查；表达更顺不应牺牲事实边界和证据编号。

## 输出检查

摘要完成后检查：

- 是否误读了原始 SRT。
- 是否引用了目标视频自己的既有摘要。
- 是否把 AI 生成摘要当成一手事实来源。
- 是否把后文、兄弟分支、坏结局或前作参考写成当前事实。
- 是否保留了说话人边界，尤其是称呼、转述、指控和追问。
